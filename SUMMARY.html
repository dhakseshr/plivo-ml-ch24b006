Model Description

Feature Engineering (56 features, all causal)

All features use audio from [0, pause_start) only — no future information.

Feature GroupFeaturesLinguistic Motivation
F0 (pitch) slope, final value, final/mean ratio, std, voiced fraction, last voiced region duration, fall composite Falling intonation = declarative statement = EOT; continued turns show level/rising pitch
Energy mean, final, ratio, slope (1.5s + 300ms), end-vs-start ratio Energy drops as speaker trails off at EOT; hold pauses show maintained energy
MFCC (13 + 13 deltas) Mean of 13 MFCC coefficients + 13 delta-MFCC from last 0.5s Spectral dynamics capture articulation changes; deltas detect rate-of-change patterns at turn boundary
Spectral Centroid slope/final/mean, rolloff slope/final/mean, spectral flux mean/final Spectral centroid and rolloff drop as voice energy shifts to lower frequencies at EOT; flux drops as speech becomes less dynamic
Turn context Pause index, pause_start time, speech context length, inter-pause interval, turn-level voiced fraction Later pauses in longer turns are more likely EOT; inter-pause interval captures rhythm
ZCR Zero-crossing rate (last 0.5s) High ZCR = fricative/unvoiced = more dynamic speech = continuation

Model Architecture

Soft-voting ensemble of two classifiers, wrapped in isotonic calibration:

  • GradientBoostingClassifier: 200 trees, max_depth=3, lr=0.08, subsample=0.8, min_samples_leaf=8
  • RandomForestClassifier: 300 trees, max_depth=6, min_samples_leaf=5, class_weight="balanced"
  • Feature preprocessing: StandardScaler
  • Probability calibration: CalibratedClassifierCV(cv=3, method="isotonic")

Trained on English + Hindi combined (496 pauses, 200 EOT, 296 hold). Cross-lingual training is essential because the hidden test set is predominantly Hindi.

LSTM Classifier (blended)

Bidirectional LSTM with attention pooling over frame-level features (F0, energy, MFCC per 10ms hop). 80-frame sequence window (last 800ms before pause). Blended with GBM+RF ensemble at weight 0.7/0.3 (grid-searched). Trained independently on same 5-fold splits; blend probabilities at inference time via predict.py.

MFCC_0 is proportional to log energy — corroborating energy as a key signal. MFCC deltas (especially higher-order) capture spectral rate-of-change, which distinguishes the abrupt spectral shift at turn end from gradual hold pauses.

Why It Beats the Baseline

The silence-only baseline fires on every pause regardless of prosodic context, forcing the agent to rely solely on a silence timer (up to 1.6s). Our model outputs a calibrated p_eot that reflects the acoustic likelihood of turn completion:

  • When p_eot is high (EOT-like acoustics), the agent can commit at a short delay (100–750ms).
  • When p_eot is low (hold-like acoustics), the agent waits, avoiding false cutoffs.
  • This lets the scorer find an operating point with lower mean delay while keeping interrupted turns ≤ 5%.

Iteration Log (from RUNLOG.md)

Run 1 — Baseline (1600 ms)

Silence-only baseline: p_eot = 1.0 for all pauses. Agent waits full 1.6s timeout.

Run 2 — Prosodic Features + GBM (160 ms English, val AUC 0.604)

Added 20 prosodic + temporal features. Trained GBM on English+Hindi combined. First meaningful improvement over silence baseline.

Run 3 — MFCC + Spectral Centroid (val AUC 0.736)

Added 13 MFCC means, 13 MFCC delta means, spectral centroid slope/mean/final, local 300ms energy slope. Validation AUC jumped from 0.604 → 0.736, indicating substantially better generalization.

Run 4 — Ensemble + Rolloff/Flux + OOF Predictions (5-fold AUC 0.701)

Added spectral rolloff and spectral flux features. Replaced single GBM with GBM+RF soft-voting ensemble with isotonic calibration. Switched to OOF cross-validation predictions for honest evaluation. Final model.pkl trained on full dataset.

Run 5 — Error Analysis + Targeted Feature Redesign (CV AUC 0.701→0.728)

Error analysis revealed first-pause EOTs under-predicted, Hindi HOLDs over-predicted. Narrowed energy and F0 feature windows to 0.5s, added speaker-normalized F0 (z-score within turn), added explicit language flag, added n_prior pauses and avg inter-pause interval. CV AUC improved from 0.701 to 0.728.

Run 6 — Bidirectional LSTM Blend (final submission)

Added Bidirectional LSTM with attention pooling trained on frame-level features (F0, energy, MFCC at 10ms hop, 80-frame window). Blended with GBM+RF ensemble at 0.7/0.3 weight (grid-searched). predict.py updated to load both models and blend probabilities at inference time. Hindi delay improved from 781ms to 745ms.

Where the Model Fails

  • Flat declaratives: Some Hindi dialects use level pitch for statements, confusing the F0-based features.
  • Short turns: < 1s of speech before a pause gives insufficient context for reliable F0 estimation, leading to zeroed-out features.
  • List enumerations: Speakers pause between list items with EOT-like acoustics but then continue.
  • Speaker variability: No speaker normalization; pitch range varies significantly across callers.
